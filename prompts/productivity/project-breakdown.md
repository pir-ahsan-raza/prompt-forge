# Break Down Any Project Into Actual Next Steps

**Use when:** You've got a vague project or goal and you need to turn it into something you can actually start on. The output shouldn't be a project plan — it should be a list of things you can do today.

---

## The Prompt

```
I'm working on: [PROJECT OR GOAL]

My current situation:
- What I have: [resources, existing code/content, team, timeline]
- What I don't have yet: [missing pieces, unknowns]
- Biggest uncertainty: [the thing you're most unsure about]

Break this down for me. I want:
1. The single most important thing to figure out or do first (and why first)
2. The 3-5 concrete tasks that unblock everything else
3. The one assumption I'm probably making that could be wrong

Don't give me a full project plan. Give me enough to get started today and know what to do next.
```

---

## Variations

**For a feature you're building:**
```
I need to build [FEATURE] for [PRODUCT/CONTEXT].

Users want to: [what users are trying to do]
What I'm working with: [existing codebase, tech stack, constraints]
Deadline/scope: [timeline or effort budget]

What should I figure out before writing a line of code? And what's the smallest thing I can ship to test the core assumption?
```

**For a project that's stalled:**
```
I started working on [PROJECT] and it's stalled. Here's where I got stuck:
[DESCRIBE WHERE YOU ARE]

What's the actual blocker? And what's the smallest next action that gets me moving again?
```

**For estimating a task:**
```
I need to estimate how long [TASK] will take.

Context: [relevant info — team size, existing code, similar past work]
What I think it involves: [your current understanding]

What am I probably underestimating? And what's a realistic range?
```

---

## Notes

The "biggest uncertainty" field is the most important one and the one people skip most often. If you don't know what you're uncertain about, that's the first thing to surface — everything else depends on it.

The "don't give me a full project plan" line exists because without it, you'll get a 12-point waterfall breakdown. That's not useful when you need to start. You can always ask for more depth on a specific area once you know where to focus.
